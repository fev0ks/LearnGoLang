## Какие числовые типы есть в Go

Целые signed:
- `int8`
- `int16`
- `int32`
- `int64`
- `int`

Целые unsigned:
- `uint8`
- `uint16`
- `uint32`
- `uint64`
- `uint`
- `uintptr`

Floating-point:
- `float32`
- `float64`

Complex:
- `complex64`
- `complex128`

## Какой максимум у `int64`

`int64` это signed 64-bit integer.

Диапазон:

```text
min = -2^63
max =  2^63 - 1
```

Максимальное значение:

```text
9223372036854775807
```

В виде степени:

```text
2^63 - 1
```

Минимальное значение:

```text
-9223372036854775808
```

В виде степени:

```text
-2^63
```

## Какой максимум у `uint64`

Диапазон:

```text
0 .. 2^64 - 1
```

Максимум:

```text
18446744073709551615
```

## Что такое стандартный `int`

`int` это signed integer размером в word size платформы.

То есть:
- на 32-bit системе `int` обычно 32-битный;
- на 64-bit системе `int` обычно 64-битный.

Практически сегодня почти всегда:
- на обычных серверах и ноутбуках `int` = 64 бита.

Но важно:
- это не гарантированно `int64`;
- размер `int` platform-dependent.

## Диапазон `int`

Если платформа 64-bit:

```text
min = -2^63
max =  2^63 - 1
```

Если платформа 32-bit:

```text
min = -2^31
max =  2^31 - 1
```

## Когда использовать `int`, а когда `int64`

`int`:
- индексы;
- длины;
- счетчики внутри процесса;
- работа с `len`, `cap`, слайсами и коллекциями.

`int64`:
- внешний контракт;
- storage schema;
- API fields;
- timestamp counters;
- ID и числовые поля, где важен фиксированный размер.

Практическое правило:
- для "обычной арифметики в коде" часто нормален `int`;
- для форматов данных и DB/API boundary лучше фиксированный размер вроде `int64`.

## Переполнение

Для integer arithmetic переполнение возможно.

Пример:

```go
var x int8 = 127
fmt.Println(x + 1) // overflow
```

Важно:
- compile-time constant overflow компилятор ловит;
- runtime arithmetic у integer types может переполниться по правилам типа.

У unsigned переполнение особенно легко увидеть:

```go
var x uint8 = 255
fmt.Println(x + 1) // 0
```

## Полезные константы

В стандартной библиотеке можно использовать:

```go
math.MaxInt8
math.MaxInt16
math.MaxInt32
math.MaxInt64
math.MaxUint8
math.MaxUint16
math.MaxUint32
math.MaxUint64
math.MaxInt
math.MinInt
```

## Короткая таблица

| Type | Bits | Signed | Range |
| --- | --- | --- | --- |
| int8 | 8 | yes | `-2^7 .. 2^7-1` |
| int16 | 16 | yes | `-2^15 .. 2^15-1` |
| int32 | 32 | yes | `-2^31 .. 2^31-1` |
| int64 | 64 | yes | `-2^63 .. 2^63-1` |
| uint8 | 8 | no | `0 .. 2^8-1` |
| uint16 | 16 | no | `0 .. 2^16-1` |
| uint32 | 32 | no | `0 .. 2^32-1` |
| uint64 | 64 | no | `0 .. 2^64-1` |
| int | 32 or 64 | yes | platform-dependent |
| uint | 32 or 64 | no | platform-dependent |
