# Time And Space Complexity

Сложность алгоритма это способ грубо оценить, как растет стоимость работы при росте входных данных.

## Самые частые обозначения

`O(1)`:
- постоянное время;
- пример: доступ к элементу слайса по индексу.

`O(log n)`:
- время растет медленно;
- пример: бинарный поиск.

`O(n)`:
- нужно пройти все элементы;
- пример: линейный поиск по слайсу.

`O(n log n)`:
- типично для эффективной сортировки.

`O(n^2)`:
- два вложенных прохода;
- быстро становится больно на больших данных.

## Space complexity

Нужно думать не только о времени, но и о памяти.

Примеры:
- in-place алгоритм может иметь `O(1)` доп. памяти;
- создание дополнительной map может дать `O(n)` по памяти.

## Простые примеры

Линейный поиск:
- time `O(n)`
- space `O(1)`

Проверка дубликатов через map:
- time `O(n)`
- space `O(n)`

Два вложенных цикла:
- time `O(n^2)`

## Почему это важно backend engineer

Алгоритмическая сложность бьет не только в задачках.

Она проявляется в:
- обработке больших payload;
- deduplication;
- ranking and sorting;
- агрегации;
- log and event processing;
- batch jobs.

## Хороший interview ответ

Сильный ответ обычно:
- называет сложность;
- объясняет trade-off по памяти;
- говорит, где это важно в реальном backend.
