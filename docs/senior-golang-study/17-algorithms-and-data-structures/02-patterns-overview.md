| Признаки задачи | Паттерн | Файл |

## Обзор каждого паттерна

### Two Pointers

Суть: использовать два индекса, движущихся по массиву или списку, чтобы избежать вложенного цикла.

Типичная сложность: O(n) по времени, O(1) по памяти.

Две основные конфигурации:
- Указатели с двух концов (opposite ends) — сходятся к центру. Применяется на отсортированном массиве: двигай левый вправо, если сумма мала, и правый влево, если велика.
- Указатели с разными скоростями (fast/slow) — fast делает два шага, slow один. Позволяет обнаружить цикл (встретились), найти середину (fast дошёл до конца) или k-й элемент с конца (разрыв между указателями равен k).

Типичные задачи: Two Sum II (sorted), 3Sum, Linked List Cycle, Middle of Linked List, Remove Nth Node From End.

### Sliding Window

Суть: поддерживать окно [left, right] над массивом или строкой, расширяя его вправо и сужая слева при нарушении инварианта.

Типичная сложность: O(n) — каждый элемент входит и выходит из окна ровно один раз.

Ключевая идея: вместо того чтобы пересчитывать агрегат (сумму, частоту) с нуля при каждом сдвиге, инкрементально обновляй его. Обычно нужна хеш-таблица для подсчёта частот или переменная для текущей суммы.

```go
left := 0
freq := make(map[byte]int)
result := 0
for right := 0; right < len(s); right++ {
    freq[s[right]]++
    // сужай окно, пока инвариант нарушен
    for freq[s[right]] > 1 {
        freq[s[left]]--
        left++
    }
    if right-left+1 > result {
        result = right - left + 1
    }
}
```

Типичные задачи: Longest Substring Without Repeating Characters, Minimum Window Substring, Maximum Sum Subarray of Size K.

### Binary Search

Суть: каждый шаг отсекает половину пространства поиска, опираясь на монотонное свойство.

Типичная сложность: O(log n).

Ключевая идея: шаблон с инвариантом `[lo, hi)` или `[lo, hi]` — выбери один и придерживайся его. Ошибки почти всегда в граничных условиях: `<=` vs `<`, `mid+1` vs `mid`.

```go
lo, hi := 0, len(nums)-1
for lo <= hi {
    mid := lo + (hi-lo)/2
    if nums[mid] == target {
        return mid
    } else if nums[mid] < target {
        lo = mid + 1
    } else {
        hi = mid - 1
    }
}
return -1
```

Binary Search on answer: вместо поиска в массиве ищешь в пространстве ответов. Условие `canDo(mid)` монотонно — при x достаточно большом всегда true, при малом — false. Двигай границу, пока не найдёшь минимальный (или максимальный) допустимый x.

Типичные задачи: Search in Rotated Sorted Array, Find First and Last Position, Koko Eating Bananas, Capacity To Ship Packages.

### Trees and Graphs

Суть: задачи на деревьях и графах в основе своей — задачи на обход. Выбор между DFS и BFS определяет, что удобнее поддерживать в памяти.

Типичная сложность: O(V + E) для графов, O(n) для деревьев.

DFS на дереве — рекурсия естественна. Постпорядковый обход нужен, когда результат узла зависит от детей (например, высота поддерева). Препорядковый — когда нужно передать информацию от корня вниз.

```go
func dfs(node *TreeNode) int {
    if node == nil {
        return 0
    }
    left := dfs(node.Left)
    right := dfs(node.Right)
    return 1 + max(left, right)
}
```

BFS на дереве — итеративный обход с очередью. Удобен, когда нужно работать с уровнями: на каждой итерации внешнего цикла обрабатываешь ровно один уровень (размер очереди в начале итерации).

Union-Find: структура для эффективного объединения множеств и проверки принадлежности. Операции `Union` и `Find` с path compression и union by rank работают за O(α(n)) ≈ O(1).

Топологическая сортировка (алгоритм Кана): считай in-degree каждого узла, добавляй в очередь узлы с нулевым in-degree, обрабатывай их и уменьшай in-degree соседей.

Типичные задачи: Maximum Depth of Binary Tree, Lowest Common Ancestor, Number of Islands, Course Schedule, Redundant Connection.

### Dynamic Programming

Суть: разбить задачу на перекрывающиеся подзадачи, запомнить результаты (memoization или tabulation) и избежать повторных вычислений.

Типичная сложность: от O(n) до O(n * m) в зависимости от количества состояний.

Два шага построения решения:
1. Определи состояние: `dp[i]` — что именно означает, например "минимальное количество монет для суммы i".
2. Напиши переход: как `dp[i]` вычисляется через предыдущие состояния.

Типичные подкатегории:
- 1D DP: Fibonacci, Climbing Stairs, House Robber.
- 2D DP (строки): Longest Common Subsequence, Edit Distance.
- Knapsack: можно взять предметы с ограничением по весу.
- Interval DP: задачи на отрезках, палиндромы.

Признак применимости: если рекурсивное решение с перебором даёт правильный ответ, но TLE — проверь, пересчитываются ли одни и те же аргументы. Если да, добавь memo.

Типичные задачи: Coin Change, Longest Increasing Subsequence, Word Break, Unique Paths, 0/1 Knapsack.

### Sorting and Heap

Суть: heap (куча) — структура данных, которая за O(log n) поддерживает доступ к минимальному (или максимальному) элементу.

Типичная сложность операций: O(log n) для push/pop, O(1) для peek.

В Go стандартная библиотека `container/heap` требует реализации интерфейса из пяти методов. На практике часто удобнее инвертировать знак для max-heap поверх min-heap реализации.

Ключевые применения:
- Top-K наибольших элементов: поддерживай min-heap размером K. Если новый элемент больше вершины — вытолкни вершину и добавь новый.
- Top-K наименьших: поддерживай max-heap размером K.
- Медиана потока: два heap — max-heap для левой половины, min-heap для правой. Балансируй размеры.
- K-way merge: положи по одному элементу из каждого списка в min-heap с указателем на источник.

Сортировка для предобработки: если задача требует O(n log n), часто правильнее сначала отсортировать входные данные, а потом применить жадный подход или два указателя.

Типичные задачи: Kth Largest Element, Find Median from Data Stream, Merge K Sorted Lists, Task Scheduler.

### Backtracking

Суть: систематический перебор с отсечением (pruning) заведомо неверных ветвей.

Типичная сложность: O(n!) для перестановок, O(2^n) для подмножеств — backtracking не улучшает теоретическую сложность, но на практике отсечения резко сокращают количество проверяемых вариантов.

Шаблон: выбери элемент, добавь в текущий путь, рекурсивно продолжи, отмени выбор (backtrack).

```go
func backtrack(start int, current []int, result *[][]int, nums []int) {
    *result = append(*result, append([]int{}, current...))
    for i := start; i < len(nums); i++ {
        current = append(current, nums[i])
        backtrack(i+1, current, result, nums)
        current = current[:len(current)-1]
    }
}
```

Pruning: перед рекурсивным вызовом проверяй, может ли текущая ветвь привести к валидному ответу. Для задач с суммой — если текущая сумма уже превысила target, дальше идти не нужно.

Типичные задачи: Subsets, Permutations, Combination Sum, N-Queens, Sudoku Solver.

### Linked List

Суть: задачи на связных списках требуют аккуратной работы с указателями. Большинство решаются in-place без дополнительной памяти.

Типичная сложность: O(n) по времени, O(1) по памяти.

Ключевые техники:
- Sentinel node (dummy head): упрощает обработку граничных случаев — не нужно отдельно обрабатывать удаление головы или вставку перед ней.
- Reverse in-place: три указателя (prev, curr, next), итеративно.
- Merge двух отсортированных списков: сравнивай головы, присоединяй меньшую, двигай указатель.
- Определение середины: fast/slow указатели.

```go
// reverse linked list
var prev *ListNode
curr := head
for curr != nil {
    next := curr.Next
    curr.Next = prev
    prev = curr
    curr = next
}
return prev
```

Типичные задачи: Reverse Linked List, Merge Two Sorted Lists, Reorder List, LRU Cache, Copy List with Random Pointer.

## Структура решения задачи на интервью

Следующий пятишаговый фреймворк помогает не паниковать и систематически прийти к решению.

**1. Clarify — уточни условие**

Задай вопросы до того, как начнёшь писать код:
- Каков диапазон n? (от этого зависит допустимая сложность)
- Какие значения могут быть в массиве? Есть ли отрицательные числа? Дубликаты?
- Что возвращать, если входные данные пустые?
- Отсортирован ли входной массив?

Ответы на эти вопросы часто указывают на паттерн напрямую.

**2. Brute force — назови наивное решение**

Озвучь самое простое решение и его сложность. Это показывает, что ты понимаешь задачу. Не нужно его кодировать — достаточно описать словами: "Два вложенных цикла, O(n^2), явно медленно."

**3. Optimize — выбери паттерн**

Объясни, почему brute force избыточен, и какой инсайт позволяет его улучшить. Ссылайся на таблицу распознавания: "Массив отсортирован и нужна пара — это Two Pointers." Оцени сложность нового решения.

**4. Code — пиши, проговаривай**

Пиши код и комментируй вслух: "Инициализирую два указателя с концов. В цикле проверяю сумму. Если сумма меньше target, двигаю левый..." Это позволяет интервьюеру следить за твоим мышлением и подсказать, если ты пошёл не туда.

**5. Test — проверь на граничных случаях**

После написания кода обязательно пройдись по следующим случаям:
- Пустой входной массив или nil
- Один элемент
- Дубликаты или все одинаковые значения
- Минимальное и максимальное значение n
- Уже отсортированный или обратно отсортированный массив (для сортировочных задач)

## Сигналы для оптимизации

Когда у тебя есть рабочий, но медленный вариант, следующая таблица помогает найти направление улучшения.

| Если в brute force видишь... | Попробуй |
|---|---|
| Два вложенных цикла по одному массиву | Two Pointers или HashMap для поиска за O(1) |
| Линейный поиск в отсортированном массиве | Binary Search |
| Рекурсия вычисляет одни и те же аргументы | Memoization (top-down DP) |
| "Лучший из K" или "K наибольших" в потоке | Heap (размером K) |
| "Все комбинации" и нет явной оптимизации | Backtracking с pruning |
| Пересчёт суммы/агрегата подмассива с нуля | Prefix Sum или Sliding Window |
| Повторяющиеся обращения к одному и тому же диапазону | Sparse Table, Segment Tree (для range queries) |
| Граф с взвешенными рёбрами, кратчайший путь | Dijkstra (неотрицательные веса) или Bellman-Ford |

### Подсказка по сложности

Если n порядка 10^5 или 10^6, решение должно быть O(n log n) или лучше. Если n порядка 10^3, может подойти O(n^2). Если n не более 20 — возможно, O(2^n) с backtracking допустимо.

| Допустимая сложность | Ориентировочный n |
|---|---|
| O(n!) | n <= 10 |
| O(2^n) | n <= 20 |
| O(n^2) | n <= 10^3 |
| O(n log n) | n <= 10^6 |
| O(n) | n <= 10^7 |
| O(log n) | n <= 10^18 |
