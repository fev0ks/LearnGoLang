## Что такое контейнер

Контейнер это единица запуска приложения.

Обычно внутри контейнера:
- один основной процесс;
- свой filesystem view;
- свои env vars;
- свои limits и runtime settings.

## Что такое Pod

`Pod` в Kubernetes это минимальная deployable unit.

Pod может содержать:
- один контейнер;
- или несколько тесно связанных контейнеров.

Именно pod, а не контейнер:
- получает IP в Kubernetes;
- живет как единица планирования;
- перезапускается и заменяется Kubernetes.

## Главная мысль

Container:
- это runtime unit.

Pod:
- это orchestration unit в Kubernetes.

## Что контейнеры внутри одного pod делят между собой

Обычно они делят:
- network namespace;
- один IP;
- localhost между собой;
- volumes pod-а.

Поэтому контейнеры в одном pod могут общаться через:

```text
localhost
```

## Зачем в pod может быть больше одного контейнера

Это нужно редко, но бывает полезно:
- sidecar для логов;
- proxy sidecar;
- helper container;
- init container для подготовки окружения.

Но practical default:
- один pod = один основной контейнер приложения.

## Чем pod отличается от VM

Pod:
- не отдельная машина;
- не отдельная ОС;
- это объект Kubernetes поверх контейнеров.

## Хороший interview ответ

Если кратко:
- контейнер запускает процесс;
- pod это Kubernetes-обертка, в которой живут один или несколько контейнеров;
- pod получает network identity и является единицей деплоя и scheduling.
